# Contributing

Miramath への貢献を歓迎します。

## 1. 事前確認

- このプロジェクトはベータ版で、単一ユーザー・ローカル実行前提です。
- 大きな仕様変更は、先に Issue で提案してください。

## 2. 開発手順

1. Issue を作成または既存 Issue を確認
2. Fork / Branch 作成
3. 実装とテスト
4. PR 作成（テンプレートに沿って記載）

## 3. コーディング方針

- 変更は小さく分割し、目的を明確にする
- 既存 API の互換性を意識する
- エラーメッセージは利用者にとって説明可能にする

## 4. チェック項目

- backend: `pytest` が通ること
- frontend: `npm run lint` と `npm run build` が通ること
- README/ドキュメント差分が必要なら更新すること

## 5. セキュリティ

脆弱性報告は公開 Issue ではなく `SECURITY.md` の手順に従ってください。
